Installation

Test Parser is packaged via distutils. To install, run setup.py install. I'm currently only using source distributions, but I'll try to add installers/packages later (maybe.)

Test Parser's installation is defaulted to install all documentation to /usr/share/doc/Test_Parser. Because of the directory location, you'll need to run the install as root. Or you can change the directory to somewhere else. To change the directory, either edit setup.cfg or pass in --install-data = <new_directory>. Windows users will have to change the directory because the default is designed for *nix systems.

Requirements

  • python 3.x

  • PyQt4 (Qt Views)

  • Tkinter (TK Views)

  • PLY 3.3 (JUnit)

If you do not wish to use the features that depend upon a library, you do not need to install it.

PyQt4 installation

Because there wasn't a PyQt package available for python3 when I was developing I had to compile and install PyQt4 manually. If when you're trying to install Test Parser check to see if there is a package available. It'll be something like: python3-Qt4.

I followed the instructions found here. To quote:

 

Download sip and pyqt from ... the links on the website: http://www.riverbankcomputing.co.uk/news

Extract the .tar.gz files to, say, your home folder, yielding the following folders:

PyQt-x11-gpl-4.6.2
sip-4.9.3

Follow the install directions. That is, in each folder, run: 

python3.1 configure.py
make
sudo make install
 
 --Edward

Tkinter installation

Tkinter is usually packaged with the python distribution. *nix users will probably want to get it from their favorite repository. Windows users will probably install Tkinter during their python installation, but I don't know for sure.

PLY installation

PLY (Python Lex Yacc) is a python package that can be installed in the typical ways. Just download and run setup.py install. Make sure to install it with the correct python version, the correct version being the one used by TestParser (see above.)
