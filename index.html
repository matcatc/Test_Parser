Introduction

Test Parser is used to help visualize the test results from various test frameworks.

It is designed it to be extensible so that new test frameworks and/or user interfaces can be easily added at any point in time. This way, whenever you end up working with a new framework that does not provide a nice-clean interface, you only have to write a parser to utilize what has been already written previously. Test Parser is intended for a primarily developer audience, but I can see it being used in other situations.

History

I built Test Parser to help me visualize/read the test results from Boost's Test framework, hence its original name: Boost Test Parser. But it is no longer limited to just the Boost Test Framework and has never been affiliated with Boost, which is why I changed the name to Test Parser. And yet again I'm considering changing the name.
